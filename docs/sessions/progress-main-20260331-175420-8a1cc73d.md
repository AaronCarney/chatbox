## Key Design Decisions
- **Child safety is 90% of this project.** All LLM calls go through `/api/chat` server pipeline.
- **OWASP Top 10 LLM**: All 10 covered. Risk 1 (Prompt Injection): schema validation + random-salt delimiters + system prompt hierarchy. Risk 2: PII stripping all roles. Risk 10: 8K input cap + 1024 max_tokens + rate limiting.
- **Sidebar**: ChatBridge branding + Clerk UserButton + New Chat only. All Chatbox routes/settings/features removed.
- **Tool call flow**: user → server → OpenAI → tool_call_start SSE → frontend processes → addToolResult → continueAfterToolCalls → server → OpenAI → text response. Assistant messages with tool_calls stored in OpenAI format (`{ type: "function", function: { name, arguments } }`).
- **DOS games**: js-dos v8 CDN loaded on-demand (not at page load) to avoid blocking catalog render. Game catalog always renders first.
