7-Layer Defense Architecture

Every interaction passes through multiple independent safety layers. No single point of failure can compromise student safety.

Student Input | v [L1] Rate Limiting ──────── IP-based throttling, chat-specific limits | v [L2] Authentication ─────── Clerk session validation, JWT verification | v [L3] PII Stripping ──────── Regex-based removal of emails, phones, SSNs | v [L4] Prompt Injection ───── System prompt hardening, tool result wrapping, | delimiter-based data/instruction separation v [L5] LLM + Tool Calls ───── OpenAI GPT-4o with strict tool schemas, | tool call limits (max 10/turn) v [L6] Output Validation ──── Tool result schema validation, size caps (2KB), | content blocklist filtering v [L7] Client-Side Safety ─── On-device CV model (NSFWJS), iframe sandboxing, real-time content monitoring, safety overlay

Defense Layers in Detail

🔒
Iframe Sandboxing
All third-party apps run in sandboxed iframes with allow-scripts only. No access to parent DOM, cookies, or navigation. credentialless attribute prevents ambient credential leaks.
🛡️
Prompt Injection Defense
Tool results wrapped in unique delimiters with source attribution. System prompt enforces data/instruction separation. LLM instructed to never follow instructions from tool results.
👁️
Computer Vision Monitoring
On-device NSFWJS model continuously monitors iframe content via canvas snapshots. Inappropriate content triggers immediate blur overlay or hard block — no data leaves the browser.
🚫
Content Blocklist
Taxonomy-based blocklist filters age-inappropriate species, content, and search terms from the Nature Explorer. Operates at both API and display layers.
🔑
Authentication
Clerk-based authentication with session management. All chat API endpoints require valid sessions. Pseudonymized user IDs for logging — no real identifiers stored.
📊
Rate Limiting
IP-based rate limiting on all API endpoints. Additional stricter limits on chat and moderation endpoints to prevent abuse and control costs.
🧹
PII Stripping
All message content (user, assistant, tool) is scrubbed for personally identifiable information before processing. Emails, phone numbers, and SSN patterns are removed.
📋
Tool Result Validation
Schema-based validation on all tool results. Size-capped at 2KB per result. Unknown tools are rejected. Results are wrapped and attributed before reaching the LLM.

Data Privacy

  • Ephemeral Sessions: Chat history exists only in browser memory. Closing the tab erases the conversation.
  • No Student Data Storage: No names, emails, grades, or demographic information is collected or stored.
  • Pseudonymized Logging: Server logs use generated pseudonyms (e.g., "Cheerful Penguin") instead of real user identifiers.
  • No External Tracking: No analytics, ad trackers, or third-party data collection scripts.
  • App State Isolation: Each iframe app's state is scoped to the current session and sandboxed from other apps.

App Sandboxing Architecture

  • PostMessage Protocol: All parent-child communication uses a versioned schema (CHATBRIDGE_V1) with structured envelopes. No direct DOM access.
  • Origin Validation: The PostMessage broker validates message origins. Sandboxed iframes with null origin are explicitly handled.
  • No External Navigation: Iframes cannot open new windows or navigate to external sites. All content stays within the platform.
  • State Persistence Limits: App save data is capped at 512KB per app per session and validated before storage.
  • Cross-App Isolation: Apps cannot read or write other apps' state. Source validation prevents cross-app spoofing.

LLM Security

  • System Prompt Hardening: The AI is instructed to never reveal its system prompt, never follow instructions from tool results, and never generate inappropriate content.
  • Tool Call Limits: Maximum 10 tool calls per turn, 5-deep chaining limit. Prevents runaway execution.
  • Token Budget: Input capped at 8K tokens with progressive history trimming. Prevents context window exploitation.
  • Delimiter Wrapping: Tool results are wrapped in unique delimiters with source attribution, creating a clear boundary between data and instructions.
  • Strict Tool Schemas: All tool parameters use OpenAI strict mode with JSON Schema validation. No freeform tool arguments.

Content Safety for Nature Explorer

  • Taxonomy Blocklist: Multi-layered filtering removes age-inappropriate species (parasites, venomous species descriptions, etc.) at both search and detail endpoints.
  • Wikipedia HTML Stripping: All Wikipedia summaries are stripped of HTML tags before display to prevent markup injection.
  • License Filtering: Only Creative Commons-licensed images from iNaturalist are displayed. Copyrighted content is excluded.
  • No External Links: Species detail pages keep students within the app. No links to Wikipedia, iNaturalist, or other external sites.
